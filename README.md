## Обзор проекта

**RoadMate** — это платформа для поиска попутчиков, которая позволяет пользователям:

- 🔍 Искать поездки между городами
- 🚗 Создавать поездки в качестве водителя
- 🎫 Оставлять заявки на присоединение к поездкам в качестве пассажира
- 💬 Общаться с другими пользователями через встроенный чат
- ⭐ Оценивать и оставлять отзывы о пользователях
- 🔔 Получать уведомления об обновлениях поездок и заявок
- 👤 Управлять профилями и просматривать историю поездок

## Установка и запуск

### Клонирование репозитория

Если вы получили проект из репозитория, клонируйте его:

```bash
git clone <URL-репозитория>
cd BlaBlaCar
```

Если вы открыли проект локально, перейдите в директорию проекта:

```bash
cd <путь-до-проекта>/BlaBlaCar
```

---

### Установка бэкенда

#### Шаг 1: Перейдите в директорию бэкенда

```bash
cd backend
```

#### Шаг 2: Проверьте версию Python

```bash
python --version
# Ожидаемый вывод: Python 3.10.x или выше
```

#### Шаг 3: Создайте виртуальное окружение

**Windows (командная строка cmd.exe):**
```bash
python -m venv venv
```

**Windows (PowerShell):**
```powershell
python -m venv venv
```

**Linux/MacOS:**
```bash
python3 -m venv venv
```

#### Шаг 4: Активируйте виртуальное окружение

**Windows (командная строка cmd.exe):**
```bash
venv\Scripts\activate
```

**Windows (PowerShell):**
```powershell
.\venv\Scripts\Activate
```

**Linux/MacOS:**
```bash
source venv/bin/activate
```

После активации вы увидите `(venv)` в начале командной строки:
```
(venv) C:\Users\...\backend>
```

#### Шаг 5: Установите зависимости Python

```bash
pip install -r requirements.txt
```

**Ожидаемый вывод:**
```
Successfully installed [список пакетов]
```

#### Шаг 6: (Опционально) Настройте переменные окружения

Создайте файл `.env` в директории `backend/` со следующим содержимым:

```env
# Настройки приложения
APP_NAME=Ride Sharing API
DEBUG=True

# Настройки JWT (измените SECRET KEY в production!)
JWT_SECRET_KEY=your-secret-key-change-in-production
JWT_ALGORITHM=HS256
JWT_ACCESS_TOKEN_EXPIRE_MINUTES=1440

# CORS
ALLOWED_ORIGINS=["http://localhost:5173","http://localhost:3000"]

# База данных (False = in-memory, True = PostgreSQL)
USE_POSTGRESQL=False
```

## Запуск проекта

### Запуск бэкенда

#### Способ 1: Используя uvicorn напрямую

```bash
cd backend
uvicorn app.main:app --reload --port 8000
```

#### Способ 2: С активацией виртуального окружения

```bash
cd backend
venv\Scripts\activate
uvicorn app.main:app --reload --port 8000
```

#### Способ 3: С указанием хоста

```bash
cd backend
uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

**Ожидаемый вывод при успешном запуске:**
```
INFO:     Uvicorn running on http://127.0.0.1:8000
INFO:     Application startup complete
INFO:     Press CTRL+C to quit
```

### Запуск фронтенда

Откройте **новый терминал** (бэкенд должен оставаться запущенным):

```bash
cd frontend
npm run dev
```

**Ожидаемый вывод:**
```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

- Регистрация пользователей с email

- Создание личных чатов с другими пользователями
- Отправка и получение сообщений

### ⭐ Отзывы
- Оценка пользователей после поездки (1-5 звёзд)
- Текстовые отзывы

### 🔔 Уведомления

- Статус прочитано/непрочитано

## Устранение проблем

### Порт занят

**Бэкенд (порт 8000):**
```bash
uvicorn app.main:app --reload --port 8001
```

**Фронтенд (порт 5173):**
```bash
npm run dev -- --port 5174
```

### Ошибка "python не является внутренней или внешней командой"

Добавьте Python в переменную PATH или используйте полный путь к python.exe.

### Ошибка при установке зависимостей Python

```bash
# Обновите pip
python -m pip install --upgrade pip

# Переустановите зависимости
pip install -r requirements.txt
```

### Ошибка при установке зависимостей Node.js

```bash
# Очистите кэш
npm cache clean --force

# Переустановите
rm -rf node_modules package-lock.json
npm install
```

### Бэкенд не запускается

1. Проверьте, что виртуальное окружение активировано `(venv)`
2. Проверьте, что все зависимости установлены `pip list`
3. Проверьте файл `.env` на наличие ошибок

### Фронтенд не запускается

1. Проверьте, что Node.js установлен `node --version`
2. Проверьте, что зависимости установлены `npm install`
3. Проверьте порт 5173 не занят

### Ошибка CORS

Добавьте адрес фронтенда в `ALLOWED_ORIGINS` в файле `.env` и перезапустите бэкенд.

### Виртуальное окружение не активируется (Windows PowerShell)

```powershell
# Запустите PowerShell от имени администратора и выполните:
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

После запуска бэкенда откройте:

### Основные эндпоинты

| Категория | Эндпоинт | Описание |
|-----------|----------|----------|
| **Аутентификация** | POST /api/auth/register | Регистрация |
| | POST /api/auth/login | Вход |
| | GET /api/auth/me | Текущий пользователь |
| **Поездки** | POST /api/trips | Создать поездку |
| | GET /api/trips | Поиск поездок |
