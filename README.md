## Обзор проекта

**RoadMate** — это платформа для поиска попутчиков, которая позволяет пользователям:

- 🔍 Искать поездки между городами
- 🚗 Создавать поездки в качестве водителя
- 🎫 Оставлять заявки на присоединение к поездкам в качестве пассажира
- 💬 Общаться с другими пользователями через встроенный чат
- ⭐ Оценивать и оставлять отзывы о пользователях
- 🔔 Получать уведомления об обновлениях поездок и заявок
- 👤 Управлять профилями и просматривать историю поездок

## Быстрый старт

Если у вас уже установлены Python и Node.js, выполните следующие шаги:

### 1. Клонируйте репозиторий (если есть) или перейдите в директорию проекта

```bash
cd BlaBlaCar
```

### 2. Запустите бэкенд

```bash
# Windows
cd backend
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

### 3. Запустите фронтенд (в новом терминале)

```bash
cd frontend
npm install
npm run dev
```

### 4. Откройте браузер

Перейдите по адресу: **http://localhost:5173**

## Подробная установка

### Установка бэкенда

#### Шаг 1: Перейдите в директорию бэкенда

```bash
cd backend
```

#### Шаг 2: Создайте виртуальное окружение Python

**Windows (cmd.exe):**
```bash
python -m venv venv
```

**Windows (PowerShell):**
```powershell
python -m venv venv
```

**Linux/MacOS:**
```bash
python3 -m venv venv
```

#### Шаг 3: Активируйте виртуальное окружение

**Windows (cmd.exe):**
```bash
venv\Scripts\activate
```

**Windows (PowerShell):**
```powershell
.\venv\Scripts\Activate
```

**Linux/MacOS:**
```bash
source venv/bin/activate
```

После активации вы увидите `(venv)` в начале командной строки.

#### Шаг 4: Установите зависимости Python

```bash
pip install -r requirements.txt
```

#### Шаг 5: Настройте переменные окружения (опционально)

Создайте файл `.env` в директории `backend/` со следующим содержимым:

```env
# Настройки приложения
APP_NAME=Ride Sharing API
DEBUG=True

# Настройки JWT (измените SECRET KEY в production!)
JWT_SECRET_KEY=your-secret-key-change-in-production
JWT_ALGORITHM=HS256
JWT_ACCESS_TOKEN_EXPIRE_MINUTES=1440

# CORS - разрешённые источники
ALLOWED_ORIGINS=["http://localhost:5173","http://localhost:3000"]

# База данных ( False = in-memory, True = PostgreSQL )
USE_POSTGRESQL=False
```

#### Шаг 6: Запустите бэкенд

```bash
uvicorn app.main:app --reload --port 8000
```

Вы должны увидеть примерно следующее:
```
INFO:     Uvicorn running on http://127.0.0.1:8000
INFO:     Application startup complete
```

## Основные функции

### 🔐 Аутентификация
- Регистрация пользователей с email
- Безопасный вход с JWT токенами
- Поддержка refresh токенов
- Хэширование паролей с bcrypt
- Защищённые маршруты

### 🚗 Поездки
- Создание новых поездок (в качестве водителя)
- Поиск поездок по месту отправления, назначения и дате
- Просмотр деталей поездки
- Обновление и удаление поездок
- Управление статусом поездки (активна, завершена, отменена)
- Просмотр своих поездок как водитель

### 🎫 Заявки на поездки
- Отправка заявок (в качестве пассажира)
- Просмотр ожидающих заявок (в качестве водителя)
- Принятие или отклонение заявок
- Отслеживание статуса заявки
- Обработка параллельных заявок

### 💬 Чат
- Создание личных чатов с другими пользователями
- Отправка и получение сообщений
- Просмотр истории переписки
- Индикация прочтения сообщений

### ⭐ Отзывы и рейтинг
- Оценка пользователей после поездки (1-5 звёзд)
- Текстовые отзывы
- Просмотр отзывов о пользователе
- Автоматический расчёт среднего рейтинга

### 🔔 Уведомления
- Внутриприложенные уведомления
- Статус прочитано/непрочитано
- Уведомления о новых заявках
- Уведомления об изменении статуса заявки
- Уведомления о новых сообщениях и отзывах

### 👤 Профиль
- Просмотр профилей пользователей
- Редактирование своего профиля
- Просмотр истории поездок
- Просмотр рейтинга и отзывов

## Конфигурация

### Переменные окружения бэкенда

Создайте файл `.env` в директории `backend/`:

```env
# Настройки приложения
APP_NAME=Ride Sharing API
DEBUG=True

# Настройки JWT
# ВАЖНО: Измените SECRET KEY в production!
JWT_SECRET_KEY=super-secret-key-change-me
JWT_ALGORITHM=HS256
JWT_ACCESS_TOKEN_EXPIRE_MINUTES=1440

# CORS - разрешённые источники (фронтенд)
ALLOWED_ORIGINS=["http://localhost:5173","http://localhost:3000"]

# База данных
# False = in-memory (для разработки)
# True = PostgreSQL (для production)
USE_POSTGRESQL=False

# PostgreSQL настройки (только если USE_POSTGRESQL=True)
# POSTGRES_HOST=localhost
# POSTGRES_PORT=5432
# POSTGRES_USER=postgres
# POSTGRES_PASSWORD=your_password
# POSTGRES_DB=blablacar
```

### Переключение между In-memory и PostgreSQL

По умолчанию проект использует **In-memory хранилище**, которое хранит данные в памяти и сбрасывается при перезагрузке сервера. Это идеально для разработки.

Для использования **PostgreSQL**:
1. Установите и запустите PostgreSQL
2. Создайте базу данных `blablacar`
3. Измените `USE_POSTGRESQL=True` в `.env`
4. Укажите параметры подключения к PostgreSQL

## Устранение проблем

### Порт уже занят

Если порт 8000 или 5173 уже занят другим приложением:

**Бэкенд:**
```bash
uvicorn app.main:app --reload --port 8001
```
Затем используйте `http://localhost:8001`

**Фронтенд:**
```bash
npm run dev -- --port 5174
```
Затем используйте `http://localhost:5174`

### Ошибки установки Python зависимостей

```bash
# Обновите pip
python -m pip install --upgrade pip

# Переустановите зависимости
pip install -r requirements.txt
```

### Ошибки установки Node.js зависимостей

```bash
# Очистите кэш npm
npm cache clean --force

# Переустановите зависимости
rm -rf node_modules package-lock.json
npm install
```

### Бэкенд не отвечает

1. Убедитесь, что бэкенд запущен
2. Проверьте терминал на наличие сообщений об ошибках
3. Убедитесь, что бэкенд отображает:
   ```
   INFO:     Uvicorn running on http://127.0.0.1:8000
   ```

### Ошибки CORS

Если фронтенд не может связаться с бэкендом:
1. Проверьте, что адрес фронтенда добавлен в `ALLOWED_ORIGINS` в `.env`
2. Перезапустите бэкенд после изменения `.env`

### Виртуальное окружение не активируется (Windows)

Если возникает ошибка "cannot be loaded because running scripts is disabled":
```powershell
# Запустите PowerShell от администратора и выполните:
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

## Технологический стек

- **Zustand** — Управление состоянием
- **CSS Modules** — Стилизация

- **Паттерн Репозиторий** — Абстракция доступа к данным
